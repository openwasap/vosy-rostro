## Constatación Crítica: Cámara Virtual en el Mismo Teléfono

### Requisito del usuario: WhatsApp debe usar la cámara de esta app

**Android permite cámara virtual exportable a WhatsApp desde Android 14 (API 34) mediante VirtualCameraService**, pero con restricciones importantes:

| Requisito | Detalle |
|---|---|
| **API level** | Android 14 (API 34) mínimo |
| **Tipo de app** | Debe ser app de sistema preinstalada (`privileged` app) |
| **Firma** | Debe estar firmada con la clave del OEM o del firmware del dispositivo |
| **Instalación** | Requiere `/system/priv-app/` o `/system_ext/priv-app/` |
| **Permiso** | `android.permission.CREATE_VIRTUAL_CAMERA` (signature) |

**Sin root / sin ser app de sistema**: NO es posible crear una cámara virtual que WhatsApp detecte y use.

### Rutas factibles para el usuario

#### Ruta A: Root + Magisk + Módulo Xposed (COMPROBADA)
- Instalar Magisk (root) + LSPosed
- Usar módulo XVirtualCamera (https://github.com/sandyz987/XVirtualCamera)
- El módulo crea un dispositivo de cámara virtual exportable
- **WhatsApp lo detecta y se puede seleccionar en llamadas** (confirmado por reportes)
- La app Decart AI puede alimentar el módulo con frames procesados por Decart AI mediante endpoint local HTTP/RTSP
- **Complejidad**: Alta (requiere root, bootloader desbloqueado)

#### Ruta B: App de sistema con VirtualCameraService (SIN root pero requiere build propio de firmware)
- Compilar AOSP con la app preinstalada en `/system/priv-app/`
- La app declara `VirtualCameraDevice` en `CameraManager`
- **WhatsApp la detecta automáticamente** como cámara hardware
- **Complejidad**: Muy alta (requiere build de firmware, no apto para usuarios finales)

#### Ruta C: Flujo actual WebRTC → PC → SplitCam (SIN modificaciones complejas)
- Móvil envía video a PC vía WebRTC
- PC aplica filtros Decart AI
- SplitCam captura la ventana del navegador y crea cámara virtual para WhatsApp
- **WhatsApp en el PC usa la cámara virtual de SplitCam**
- **Complejidad**: Baja (ya implementado)

## 3. Lo que NO se puede hacer (y alternativas)

### 3a. NO se puede: Cámara virtual exportable a WhatsApp SIN root ni ser app de sistema
**Por qué**: `CREATE_VIRTUAL_CAMERA` es permiso `signature|privileged`. Solo apps en `/system/priv-app/` pueden registrar `VirtualCameraDevice`.

**Sí se puede con root**: Usar Magisk + XVirtualCamera. La app alimenta el módulo con frames procesados.

**Sí se puede con PC intermedio**: Arquitectura actual WebRTC → PC → SplitCam.

### 3b. NO se puede: Hacer que la app exporte cámara virtual sin root/SDK 34+
Android 15+ introduce APIs para virtual camera device creation, pero requeriría ser app preinstalada de sistema.

### Flujo de audio bidireccional

```
APK (micrófono) ──WebRTC──► PC (altavoces)
PC (micrófono) ──WebRTC──► APK (altavoces/auriculares)
```

Ambos dispositivos deben:
1. Activar `echoCancellation: true`, `noiseSuppression: true` en `getUserMedia`
2. Usar `AudioContext` o elementos `<audio>` para reproducción
3. Manejar el evento `ontrack` para streams de audio entrantes
4. Controlar `enabled` en tracks de audio para mutear/unmutear

| Restricción | Solución |
|---|---|
| Android no permite cámara virtual sin root/sistema | Ruta A: Root + Magisk + XVirtualCamera (confirmado que funciona con WhatsApp) |
| Permiso `CREATE_VIRTUAL_CAMERA` es `signature\|privileged` | Solo apps en `/system/priv-app/` pueden usarlo directamente |
| Sin root, no hay cámara virtual en el propio teléfono | Ruta C: WebRTC → PC → SplitCam (ya implementado) |
| WebRTC a través de NAT simétrico requiere TURN | Incluir servidor TURN en el despliegue |
| Sin cuenta Expo/EAS, no se genera APK firmado remotamente | Requiere cuenta Expo o build local con Android SDK |

## Pregunta Crítica del Usuario

**"Si se dan los permisos, ¿puede generar su propia camara? El celular esta nuevo, sin preparacion."**

### Respuesta Directa: NO, con permisos normales NO alcanza

| Permiso que el usuario PUEDE conceder | Permiso que el usuario NO puede conceder |
|---|---|
| `CAMERA` | `CREATE_VIRTUAL_CAMERA` (signature\|privileged) |
| `RECORD_AUDIO` | `CAMERA_DEVICE_CREATION` (signature) |
| `WRITE_EXTERNAL_STORAGE` | `MEDIA_PROVIDER` avanzado |
| `INTERNET` | |

El permiso `CREATE_VIRTUAL_CAMERA` está clasificado como `signature|privileged` en Android:
- **`signature`**: Solo apps firmadas con el mismo certificado que el **firmware del dispositivo** pueden obtenerlo
- **`privileged`**: Solo apps instaladas en `/system/priv-app/` o `/system_ext/priv-app/`

**No existe forma de otorgar estos permisos desde Settings > Apps > Permissions.** Ni siquiera con `adb grant` funciona porque el sistema lo rechaza para apps normales.

### Comparación de Rutas

| Ruta | ¿Con permisos normales alcanza? | ¿Root? | ¿WhatsApp detecta cámara? |
|---|---|---|---|
| Permisos normales + APK standalone | NO | No | NO |
| Root + Magisk + LSPosed + XVirtualCamera | No (pero el módulo sí tiene el permiso como sistema) | Sí | SÍ |
| WebRTC → PC → SplitCam | N/A (no depende del teléfono) | No | SÍ (en el PC) |

### Conclusión

Con el teléfono nuevo, sin root, sin Magisk, sin ser app de sistema: **NO ES POSIBLE** que WhatsApp detecte una cámara virtual generada por esta app.

El usuario tiene exactamente **2 opciones viables**:
1. **Rootear el teléfono** e instalar Magisk + LSPosed + XVirtualCamera (APK sirve frames por HTTP)
2. **Usar la arquitectura PC**: APK envía video por WebRTC al PC, SplitCam crea la cámara virtual, WhatsApp la usa en el PC

1. Desplegar el backend (`artifacts/api-server`) en un servidor accesible (Replit, Railway, Render, VPS)

3. Instalar un servidor TURN (coturn) para conexiones remotas

1. Desplegar el backend en un servidor accesible
